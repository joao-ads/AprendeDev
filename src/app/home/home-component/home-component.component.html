Primeiro passo Instalação do Node.Js

Instalação do Node.JS indo para o site https://nodejs.org/en/ baixe a versão compativel ao seu sistema operacional, recomendavel fazer o download e instalação da versão LTS.
Se você não tiver certeza de qual versão do Node.js é executada em seu sistema, execute node -v em uma janela de seu terminal(cmd ou Ide de sua preferencia).







Segundo Passo instalação do Angular CLI

Com Node.Js intalado teremos gerenciador de pacotes npm, com ele faremos a instação do Angular CLI.
Você usa a CLI Angular para criar projetos, gerar código de aplicativo e biblioteca e executar uma variedade de tarefas de desenvolvimento em andamento, como teste, empacotamento e implantação. Para instalar o Angular CLI, abra uma janela de terminal e execute o seguinte comando: npm install -g @angular/cli









Terceiro passo criar um espaço de trabalho e uma aplicação inicial.

Para criar um novo espaço de trabalho e uma aplicação inicial.
1° - Execute o comando CLI ng new e forneça o nome my-app conforme mostrado aqui: ng new my-app.
2° - O ng new comando solicita informações sobre os recursos a serem incluídos na aplicação inicial. Aceite os padrões pressionando a tecla Enter ou Return.
O Angular CLI instala os pacotes Angular npm necessários e outras dependências. Isso pode levar alguns minutos. A CLI cria um novo espaço de trabalho e um aplicativo Bem-vindo simples, pronto para ser executado.

Quarto passo executar a aplicação.

O Angular CLI inclui um servidor, para você construir e rodar sua aplicação localmente.
1° - Navegue até a pasta do espaço de trabalho, como
my-app.
2° - Execute o seguinte comando:
cd my-app

ng serve --open
O ng serve esse comando inicia o servidor, observa seus arquivos e reconstrói o aplicação conforme você faz alterações nesses arquivos. A opção --open(ou apenas -o) abre automaticamente seu navegador para http://localhost:4200/. Se sua instalação e configuração foram bem-sucedidas, você deverá ver uma página semelhante ao lado.

Recursos mais utilizados no Angular

Component

A maior parte do desenvolvimento quando se utiliza o framework Angular é feito nos componentes.
Cada componente define uma classe, que contém dados e lógicas da aplicação e está sempre associada a um template do HTML, onde são definidos as visualizações deste componente.
O decorator @Component() identifica a classe imediatamente como um componente e oferece o modelo e os metadados específicos dele.
Os metadados configuram, por exemplo, como o componente pode ser referenciado no HTML e também quais os serviços devem ser utilizados.

Module

Um module é um mecanismo para agrupar components, directives , pipes e services relacionados, de forma a combinar com outros módulos para criar um aplicativo. Uma aplicação angular pode ser pensada como um quebra-cabeça onde cada peça (ou cada módulo) é necessária para poder ver a imagem completa. para executar o module: ng generate module < nome da aplicação>.

Exemplo para adicionar um novo module chamado AdminModule em sua aplicação execute o seguinte comando :
ng generate module admin ou ng g m admin





Directives

As diretivas são como marcadores no elemento DOM que comunicam ao Angular para incluir um comportamento específico.

Existem três tipo de diretivas no Angular, que são: Diretivas de atributos, Diretivas estruturais e Componentes.

Diretivas de atributos: Alteram a aparência ou o comportamento de um elemento, componente ou outra diretiva, como por exemplo, NgClass e NgStyle.
Diretivas estruturais: Modificam o layout adicionando ou removendo elementos do DOM, como por exemplo, NgIf e NgFor.
Componentes: São diretivas com um modelo.


Service

serviço é uma classe que pode ser utilizada por outros componentes do projeto.
Dois detalhes importantes são: o que diferencia um serviço de outro tipo de componente e como um componente utiliza um serviço.
Para criar um serviço usando Angular CLI e o comando generate (abreviado por g) e a opção service (abreviada por s) executamos por:
ng g s < nome da aplicação>







Rotas

No Angular, muita coisa é modularizada, e as rotas não são diferentes. Ao criar uma aplicação Angular, somos indagados sobre o uso das mesmas (Would you like to add Angular Routing?), e ao responder “Yes”, já temos tudo que precisamos para o roteamento.
Na verdade, tudo que precisamos para ter rotas em nossa aplicação é um arquivo de módulo das rotas e importá-lo posteriormente no Root Module.
